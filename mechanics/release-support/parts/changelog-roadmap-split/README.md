# Changelog Roadmap Split

Use this part when release history, current direction, future work, and landing
evidence are at risk of mixing.

## Use When

- A change affects `CHANGELOG.md`, `ROADMAP.md`, a mechanic `LANDING_LOG.md`,
  `DIRECTION.md`, `PARTS.md`, or public current-state wording.
- A detailed landing history is starting to bloat a roadmap or README.

## Do Not Use When

- The edit is a local typo and does not change history, direction, status, or
  route shape.

## Route Check

Put released repository history in `CHANGELOG.md`, current/future contour in
`ROADMAP.md`, checked mechanic landings in `LANDING_LOG.md`, and operational
routes in active parts.

## Active Outputs

- Correct destination for the update.
- Short route text in entry surfaces.
- Stop-line against history/direction confusion.

## Next Route

Use `direction-surface-review` if entry docs or route maps must change too.
