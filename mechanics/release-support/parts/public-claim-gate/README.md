# Public Claim Gate

Use this part before a center sentence becomes public-facing.

## Use When

- README, profile, release note, website, generated capsule, or public docs
  language claims a capability, maturity, guarantee, release state, or support
  posture.
- A claim depends on sibling evidence, proof, runtime behavior, or ToS-authored
  meaning.

## Do Not Use When

- The sentence is purely internal and already bounded by local owner truth.
- The claim belongs fully to another repository and AoA should only link to it.

## Route Check

Ask: who owns the claim, what evidence supports it, what must not be implied,
and where proof lives if the claim is quality or regression-sensitive?

## Active Outputs

- Supportable public wording.
- Stop-line wording for what cannot be claimed.
- Proof or owner route.
- Public projection handoff if `8Dionysus` must change.

## Next Route

Route proof to `aoa-evals`, public projection to `8Dionysus`, and owner-local
truth to the owning repository.
