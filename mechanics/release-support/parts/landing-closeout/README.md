# Landing Closeout

Use this part when a wave, packet, mechanic, quest, checkpoint, or part becomes
an active landing.

## Use When

- Raw source has been distilled into active surfaces.
- A mechanic changes maturity or gains active parts.
- A closeout must explain what changed, what was verified, and what remains
  routed elsewhere.

## Do Not Use When

- The work is still exploratory and cannot name evidence or owner boundaries.
- The transition belongs to a sibling repo and the center only has a request.

## Route Check

Confirm active surfaces, provenance bridge, owner requests, validation, skipped
checks, remaining risk, and next owner route.

## Active Outputs

- Landing log entry.
- Closeout summary.
- Provenance update.
- Owner request update when needed.

## Next Route

Use `owner-handoff-packet` for external adoption and `rollback-return` when a
landing needs an explicit return path.
