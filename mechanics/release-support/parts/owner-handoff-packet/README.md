# Owner Handoff Packet

Use this part when a release-support transition needs another repository to
carry implementation, proof, routing, stats, SDK support, or public projection.

## Use When

- Center work produces a ready-to-carry owner request.
- A transition depends on a sibling owner before it can be publicly claimed.

## Do Not Use When

- The owner has already accepted and landed the slice; cite the owner receipt
  instead.
- The center can complete the transition without owner-local action.

## Route Check

Name request ID, owner repo, slice, required owner landing, proof route,
handoff surface hint, stop-line, and next action.

## Active Outputs

- Owner request update.
- Handoff card.
- Stop-line against owner acceptance claims.

## Next Route

Carry the request ID into the owner repository. Keep center status as requested
until the owner lands a receipt.
