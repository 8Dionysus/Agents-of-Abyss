# Release Runbook

Use this part for repository release execution and publication discipline.

## Use When

- Preparing a PR, tag, GitHub Release, changelog section, release body, release
  audit, or repo validation pass.
- Coordinating center release posture with federation release protocol.

## Do Not Use When

- The work is only a mechanic landing with no repository release or public
  release claim.
- Runtime deployment behavior is the real owner; route to `abyss-stack`.

## Route Check

Confirm worktree state, changelog shape, README banner if applicable,
validation, tag/release body, and postpublish audit route.

## Active Outputs

- Release checklist route.
- Validation path.
- Publication stop-line.
- Postpublish audit route.
