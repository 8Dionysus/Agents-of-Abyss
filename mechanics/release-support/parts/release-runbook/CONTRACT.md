# Release Runbook Contract

A GitHub release is complete only when local release docs, changelog, tag,
GitHub Release body, latest marker, and validation agree.

## Required Before Output

- Changelog section is canonical for release notes.
- Release verifier passes.
- Public release body does not outrun owner truth.

## Stop-lines

- No manual publication path that bypasses bounded helper/audit posture.
- No release note without changelog authority.
- No release completeness claim before postpublish checks.
