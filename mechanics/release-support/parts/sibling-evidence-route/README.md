# Sibling Evidence Route

Use this part when a release-support claim cites or depends on another
repository.

## Use When

- A center release note or public claim mentions sibling readiness, proof,
  runtime behavior, profile projection, SDK support, routing, or stats.
- The evidence exists elsewhere and must stay owner-local.

## Do Not Use When

- The claim is fully center-owned and does not depend on sibling truth.
- The sibling repository has not produced any evidence; create or update an
  owner request instead.

## Route Check

Cite the owner evidence, state what the center may claim, and name what remains
owner-local.

## Active Outputs

- Evidence route.
- Weaker center wording.
- Owner request when evidence is absent.

## Next Route

Route to the sibling repository for acceptance and to `aoa-evals` when the
claim needs proof.
