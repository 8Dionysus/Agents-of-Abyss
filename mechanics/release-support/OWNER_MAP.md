# Release Support Owner Map

Release support routes transition claims. It does not own implementation truth,
proof verdicts, public projection authority, or sibling acceptance.

| Concern | Center role | Stronger owner | Stop-line |
|---|---|---|---|
| State transition | Names the transition gate and required evidence | Owning mechanic or repository | Wording change is not state change. |
| Public claim | Names supportable language and public stop-lines | Claim owner plus `aoa-evals` for proof | The center must not publish unverified promises. |
| GitHub release | Names federation completeness and center runbook | Owner repository | Tag, changelog, and release body must match owner truth. |
| Landing closeout | Names what moved into active use | Owning mechanic | Landing log is not proof of sibling adoption. |
| Changelog history | Keeps released history in root changelog | Repository owner | Roadmap detail must not become changelog truth. |
| Direction surface | Routes current course to the right doc | Repository owner | Direction maps must not outrun owner entry docs. |
| Proof | Routes evidence and verdict needs | `aoa-evals` | Release support cannot become proof authority. |
| Derived movement | Routes release windows and summaries | `aoa-stats` | Stats must not become release truth. |
| Route ABI | Names release/public entry needs | `aoa-routing` | Routing must not author release claims. |
| Compatibility helper | Names consumer loading needs | `aoa-sdk` | SDK helpers must not become release authority. |
| Public projection | Names public profile alignment | `8Dionysus` | Profile surfaces must not outrun center or owner evidence. |
| Runtime deployment | Names release stop-lines only | `abyss-stack` | Runtime state and deployment truth are runtime-owned. |

## Owner handoff rule

If a release-support finding needs owner-local action, create or update an
owner request. Do not mark the request accepted until the owner repository
produces a receipt.
