# Growth Cycle Roadmap

## Current contour

Growth Cycle is landed as a center law and route grammar. The current package
is intentionally light: it gives agents a stage map and owner split, while
leaving implementation, proof, memory, runtime, stats, and playbook behavior to
the owning repositories.

## Next Work

- Keep owner-request packets ready to carry when sibling repositories begin
  landing local Growth Cycle surfaces.
- Watch for repeated cycle receipts that deserve a compact machine-readable
  stage vocabulary.
- Keep Growth Cycle aligned with Checkpoint, Method-growth, Distillation,
  Questbook, Antifragility, Release-support, and Recurrence without duplicating
  their active law.

## Later Work

- Add part-local schemas or examples only after real owner receipts prove the
  shape is repeated.
- Add derived visibility only through `aoa-stats` and only from reviewed
  receipts.
- Add playbook choreography only through `aoa-playbooks`, not center prose.

## When The Time Comes

Use this block for future work that is likely but not yet worth landing:

- Define a stage ID vocabulary once more than one owner repo needs compatible
  cycle state names.
- Add a Growth Cycle quest lane only when repeated cycle work becomes public
  obligation rather than private process residue.
- Add an eval bundle once public claims about repair, progression, or
  automation opportunity need defended evidence.

## Out Of Scope

- Hidden scheduler behavior.
- Autonomous self-repair claims.
- Universal progression scores.
- Owner-local hook, skill, proof, memory, runtime, stats, or playbook truth.
