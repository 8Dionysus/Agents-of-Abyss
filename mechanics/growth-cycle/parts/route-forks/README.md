# Route Forks

Route Forks makes material next choices explicit before the cycle pretends a
single path is inevitable.

## Use When

- The next step could be harvest, repair, quest, owner handoff, automation,
  memo, stats, or stop.
- Several valid owners could receive the pressure.
- A decision should name cost, risk, and stop condition.

## Do Not Use When

- The owner is already fixed by repository law.
- The fork is cosmetic and does not change action.
- The route needs a playbook-owned scenario rather than center prose.

## Route Check

Ask which choices are materially different, which evidence each requires, and
what would make each choice wrong.

## Active Outputs

- route fork list
- cost and risk note
- stop condition
- preferred next route
- owner handoff hint
