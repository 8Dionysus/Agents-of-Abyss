# Donor Harvest

Donor Harvest turns reviewed residue into owner-routed candidate pressure
without minting final object truth.

## Use When

- Repeated work produced a reusable unit after reviewed closeout.
- A skill, technique, playbook, seed, proof, memory lesson, or owner object may
  be worth carrying.
- The source still needs provenance and owner boundaries.

## Do Not Use When

- There is no reviewed source.
- The object already belongs in Method-growth as final object lineage.
- The center would need to claim executable truth or owner acceptance.

## Route Check

Ask what kind of object the donor residue wants to become, then route to the
stronger owner before claiming that identity.

## Active Outputs

- donor harvest route
- candidate owner hint
- evidence refs
- provenance note
- owner request pressure

## Next Route

`aoa-techniques`, recurring scenarios to `aoa-playbooks`, seed staging to
`Dionysus`, and final object truth to the final owner repository.
