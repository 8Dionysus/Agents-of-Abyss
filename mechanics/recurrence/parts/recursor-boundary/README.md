# Recursor Boundary

Recursor Boundary keeps future recursor witness and executor posture visible
without making recursor agents live.

## Use When

- A recurrence surface mentions recursor readiness, witness posture, executor
  posture, or projection candidates.
- A no-spawn boundary must stay explicit.
- Future actor activation needs owner review before installation.

## Do Not Use When

- The route wants to spawn, summon, run, or install recursor agents now.
- Readiness is being treated as runtime activation.
- Witness posture is being treated as scar, verdict, or memory ownership.

## Route Check

Ask whether the recursor surface is readiness, boundary check, or projection
candidate only. If it crosses into spawn or runtime, stop and require a later
explicit owner wave.

- no-spawn stop-line
- witness boundary
- executor boundary
- projection-candidate handoff
