# Memory Return

Memory return keeps stress lessons reusable without letting memory become live
truth.

## Use When

- A stress event, deletion, or repair produced a lesson worth retaining.
- Future agents need a recall route after cleanup.
- A raw source was distilled and the trace must remain findable.

## Do Not Use When

- The memory object would certify current health.
- The lesson has no owner, retention reason, or forgetting boundary.
- Proof belongs in `aoa-evals` before memory can be useful.

## Route Check

Name the lesson, source receipt, retention reason, forgetting boundary, and
recall route.

## Active Outputs

- Memory request.
- Retained lesson.
- Forgetting boundary.
- Recall route.

## Next Route

Route retained stress lessons to `aoa-memo`; route proof-dependent memory
integrity claims to `aoa-evals`.
