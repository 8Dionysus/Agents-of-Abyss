# Sprawl Control

Sprawl control asks whether a new surface earns its existence.

## Use When

- A new doc, schema, example, generated family, route hint, skill, technique,
  playbook, or helper is proposed.
- An active surface is becoming a dumping ground.
- Several surfaces repeat the same guidance.

## Do Not Use When

- A genuinely new object kind needs a source receipt or schema.
- The change is only a small route link in an existing canonical surface.
- The real issue is owner ambiguity rather than surface area.

## Route Check

Ask what becomes smaller, narrower, moved, merged, suppressed, quarantined,
deprecated, or deleted because this surface exists.

## Active Outputs

- One-in-one-out note.
- Merge or move route.
- Exemption with reason.
- Surface retirement request.

## Next Route

Route owner-local sprawl to the owning repository; route center sprawl to the
smallest canonical AoA surface that can absorb it.
