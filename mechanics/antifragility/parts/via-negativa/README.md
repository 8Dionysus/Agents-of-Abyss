# Via Negativa

Via negativa removes or narrows fragile exposure before adding compensating
structure.

## Use When

- A surface duplicates meaning or steals authority.
- A temporary bridge is becoming permanent.
- Active guidance carries stale history.
- A new protective layer is proposed before subtraction was tried.

## Do Not Use When

- The material is source evidence that must be retained.
- The owner repository has not reviewed the deletion or move.
- The issue is proof failure that belongs first in `aoa-evals`.

## Route Check

Try the pruning order: keep, merge, move, suppress, quarantine, deprecate,
delete. Name the owner and the provenance that survives.

## Active Outputs

- Merge route.
- Move route.
- Suppression or quarantine note.
- Deprecation or deletion candidate.
- Provenance-preserving cleanup handoff.

## Next Route

Route owner-local subtraction to the owner repository and record retained
lessons through `aoa-memo` when the lesson should survive.
