# ONE_IN_ONE_OUT

One-in-one-out is a pressure valve against surface sprawl.

It is not a blind quota.
It is a forcing function that asks every new tracked surface to explain which
older surface becomes smaller, narrower, merged, moved, deprecated, or
removable because of its arrival.

## Trigger

Apply this rule when proposing any new tracked:

- doc
- schema
- example
- generated summary family
- route hint family
- memory object kind
- eval bundle
- technique
- skill
- agent profile
- playbook lane
- runtime helper
- wrapper or adapter surface

## Required question

For every new tracked surface, answer:

1. what existing surface becomes redundant?
2. what surface becomes smaller or narrower?
3. if nothing changes, why is this not sprawl?
4. if this is temporary, what is the exit path?

## Acceptable paired moves

A new surface may pair with any of these:

- merge a near-duplicate into a canonical surface
- move misplaced content into its proper home
- deprecate an older wrapper, schema, or view family
- suppress a low-evidence derived surface
- quarantine a stale projection pack
- delete a duplicate that has no remaining owner-local reason to exist

## Exemptions

These are not zero-questions exemptions.
They are narrow exceptions where the paired removal may happen later:

- source receipts and provenance anchors
- required schema examples for a genuinely new object kind
- bounded proof outputs that preserve a real claim family
- short-lived migration shims with explicit expiry
- safety-critical clarification docs replacing ambiguous older guidance

If you claim an exemption, say why.

## Bad applications

Do not use one-in-one-out to:

- delete evidence you dislike
- remove active operator guidance without a better surviving home
- force symmetry where the real issue is missing ownership
- satisfy the rule by deleting something irrelevant

## Good outcome

A good one-in-one-out pass makes the surface area flatter.
The system reads more clearly after the addition than before it.
If that is not happening, the new surface probably is not earned yet.
