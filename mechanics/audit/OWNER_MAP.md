# Audit Owner Map

Audit routes what can be seen. Stronger owners decide what becomes proof, memory, runtime, release support, workflow, technique, campaign, or source-local remediation.

## Owner Routes

| Owner | Audit route |
|---|---|
| Source repository | owns the audited surface and any local remediation |
| `aoa-evals` | owns proof verdicts, regression evidence, comparison gates, and quality claims |
| `aoa-memo` | owns durable memory, recall objects, and memory writeback |
| `aoa-playbooks` | owns recurring audit campaign choreography |
| `aoa-skills` | owns executable audit workflows |
| `aoa-techniques` | owns reusable audit techniques |
| `aoa-stats` | owns derived audit observability and movement summaries |
| `mechanics/release-support` | owns release readiness, transition support, and public-claim posture |
| `mechanics/antifragility` | owns pruning pressure, deletion-candidate discipline, and via-negativa posture |
| `mechanics/checkpoint` | owns checkpoint evidence and save-state routing |
| `mechanics/distillation` | owns raw-to-active extraction when audit evidence is being distilled |
| `mechanics/growth-cycle` | owns recurring cycle lift from audit into repair, harvest, or next route |
| `abyss-stack` | owns runtime services, storage, deployment, and lifecycle |
| `aoa-sdk` | owns typed helpers, control-panel support, and local integration |
| `aoa-routing` | owns live route behavior and dispatch surfaces |
| `aoa-kag` | owns derived knowledge substrates |
| `aoa-agents` | owns role posture, handoffs, and agent-facing contracts |
| `8Dionysus` | owns public entry/profile route material |
| `Tree-of-Sophia` | owns ToS-authored meaning and canon |

## Center Stop-line

The center may name a route and carry a request packet. The target owner must still accept, land, validate, or reject the work in its own surface.
