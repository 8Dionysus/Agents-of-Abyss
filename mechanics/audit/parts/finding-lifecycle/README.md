# Finding Lifecycle

## Use When

Use this part when a signal must become a finding, owner request, supersession, closure, or follow-up route.

## Do Not Use When

Use a local issue, quest, or owner work item when the finding already belongs to an owner-local execution queue.

## Route Check

Name current state, evidence, owner, requested action, stop-line, and closure condition.

## Active Outputs

- finding packet
- request route
- supersession note
- closure note
- residual risk note

## Next Route

Route recurring finding campaigns to `aoa-playbooks`; route owner-local closure to the source owner.
