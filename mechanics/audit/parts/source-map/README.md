# Source Map

## Use When

Use this part when an audit needs to locate the surface, owner, authority level, source chain, and generated or legacy status before naming a finding.

## Do Not Use When

Use the target owner repository directly when ownership is already settled and the work is owner-local.

## Route Check

Start with source path, nearest `AGENTS.md`, source-of-truth status, generated status, and owner repo.

## Active Outputs

- source map
- owner boundary note
- authority level note
- generated or legacy status note
- next owner route

## Next Route

Route unclear ownership to `docs/FEDERATION_RULES.md` and `docs/REPO_ROLES.md`; route owner-local findings to the source owner.
