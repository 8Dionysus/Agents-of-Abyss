# Risk Signal

## Use When

Use this part when ambiguity, drift, cleanup pressure, fragility, duplicated authority, stale pathing, or unchecked claims need to become routeable.

## Do Not Use When

Use the target owner directly when the risk is already accepted by that owner and has an active remediation path.

## Route Check

Name the visible risk, impacted surface, likely owner, and what would reduce uncertainty.

## Active Outputs

- risk signal
- ambiguity note
- drift note
- cleanup pressure
- owner route

## Next Route

Route pruning pressure to `mechanics/antifragility`; route source-local repair to the owning repository.
