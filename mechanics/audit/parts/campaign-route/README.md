# Campaign Route

## Use When

Use this part when multiple surfaces need coordinated audit review across one theme, owner boundary, release, mechanic, or repository lane.

## Do Not Use When

Use a single finding route when one bounded surface is enough.

## Route Check

Name campaign scope, included surfaces, owner split, finding lifecycle, and campaign closeout.

## Active Outputs

- campaign route
- surface set
- owner split
- campaign finding index
- closeout route

## Next Route

Route recurring campaign choreography to `aoa-playbooks`; route campaign work items to `mechanics/questbook` when they must survive the current diff.
