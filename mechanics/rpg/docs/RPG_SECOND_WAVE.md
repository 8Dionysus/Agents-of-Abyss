# RPG Second Wave

## Role

This document defines the second bounded rollout for the AoA RPG reflection layer.

It is a second-wave contour, not a claim that the whole RPG layer is already mature.

## Goal

Prove that AoA can extend the RPG reflection layer into skills and techniques without:

- mutating skill canon
- mutating technique canon
- collapsing playbooks into build chatter
- collapsing pack profiles into runtime inventory
- collapsing technique readiness into a feat shortcut
- widening into routing or runtime mechanics too early

## Included in this wave

### `Agents-of-Abyss`
Owns:
- the common second-wave model
- the second-wave scope note
- federation-level boundary wording

### `aoa-techniques`
Owns:
- technique feat / perk reflection
- mastery-harvest posture
- feat eligibility wording tied to existing canon and evidence

### `aoa-skills`
Owns:
- skill ability-card reflection
- loadout posture grounded in existing pack profiles
- ability unlock hints tied to first-wave progression axes

### `Dionysus`
Owns:
- seed-garden prep-pack staging
- transport and lineage reflection
- no doctrinal takeover

- `aoa-kag`

## Success criteria

The second wave is successful when:

- skill abilities are derived from skill bundles instead of hand-authored RPG rewrites
- pack profiles are treated as loadout posture hints instead of runtime inventory
- technique feats are derived from canon, readiness, and evidence instead of bypassing review
- ability and feat unlock posture reuses first-wave progression axes
- `Dionysus` can stage the pack as transport and lineage, not canon

## Failure signs

Pause the contour if:

- `SKILL.md` starts collecting volatile RPG metadata as primary meaning
- `TECHNIQUE.md` starts collecting feat doctrine as primary meaning
- playbooks are pressured into generic build chatter instead of scenario method
- routing starts presenting live ability-selection authority
- runtime or UI assumptions sneak into repos that do not own runtime
