# AGENTS.md

## Applies to

This card applies to `mechanics/rpg/legacy/` and its descendants.

## Role

RPG legacy keeps historical wave sources, migration receipts, and artifact provenance.
It is a reading lane for audit and recovery, not the active mechanic route.

## Read before editing

Read `mechanics/rpg/AGENTS.md`, `mechanics/rpg/PROVENANCE.md`, and `mechanics/rpg/legacy/INDEX.md` before changing legacy surfaces.

## Boundaries

- Do not cite `legacy/raw/` as current RPG law without going through `mechanics/rpg/PROVENANCE.md`.
- Do not add new active doctrine here; distill it into `parts/`, `DIRECTION.md`, `ROADMAP.md`, or an owner repository.
- Preserve moved-source identity when historical filenames matter.

## Validation

Use the package validation lane in `mechanics/rpg/AGENTS.md`.

## Closeout

Report which raw source was consulted, which active part received the distillation, and whether any raw-only residue remains.
