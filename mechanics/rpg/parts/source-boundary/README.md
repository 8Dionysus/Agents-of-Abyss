# Source Boundary

This part owns the active RPG source-boundary rule: RPG reflects existing AoA surfaces, but source owners keep meaning.

The repository that owns the source object keeps the meaning. RPG may add a reading only when that reading stays visibly derived.

## Use When

- an RPG label might be mistaken for source truth
- a route needs the source/proof/runtime/presentation precedence made explicit
- a role, skill, proof, quest, campaign, memory, or runtime claim needs a stronger owner route

## Do Not Use When

- the task belongs wholly inside the source owner and needs no RPG reading
- the proposed RPG layer would create hidden ontology or a universal power score
- presentation language would obscure canonical machine keys or source refs

## Precedence

1. source meaning wins
2. proof and witness constrain claims
3. runtime/session state carries current execution only
4. presentation/theme labels remain derived

## Route Check

- Source object: identify the authored source, generated mirror, runtime state, proof object, or presentation label.
- Owner route: keep the stronger owner closer than the RPG reflection.
- Reading value: allow RPG language only as a visible reflection map.
- Closeout: record which owner keeps meaning and what RPG did not claim.

## Active Outputs

- source-boundary reminder
- owner route for the underlying claim
- non-authoritative reflection map proposal
