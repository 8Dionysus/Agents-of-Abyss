# ToS Soil Prep Support

This document records the current eighth-wave AoA doctrine for supporting ToS soil work before manual corpus movement resumes.

The point is narrow:
AoA may help route and support preparation.
It does not author ToS tree motion, node meaning, or expansion claims.

## Core rule

`Tree-of-Sophia` owns:

- context-node template meaning
- manual corpus-entry gating
- pre-expansion soil doctrine
- the judgment of when wider world-thought expansion is actually active

AoA may assist with:

- routing readers into the correct ToS surfaces
- keeping support reviewable and non-invasive
- preserving ownership boundaries while preparation work is underway

## What this support does not do

This support should not:

- rename soil preparation into active expansion
- restate ToS corpus meaning as if the center owned it
- treat template or review work as if a new family had already opened
- bypass maintainer-led manual source-entry gates

## Current wave boundary

The current gate remains manual Zarathustra entry.

Until that gate opens, AoA support stays narrow:

- point to ToS-owned preparation surfaces
- keep the support role explicit
- avoid importing KAG, routing, or runtime work into the wave
