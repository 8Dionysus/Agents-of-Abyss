# Experience v0.4 Certification Forge


v0.4 turns the v0.3 pilot loop into a controlled release discipline.

v0.3 proved that receipts can flow through recurrence, candidates, verdicts, memory gates, owner dispatch and assistant projection.
v0.4 asks the harder question: when may a reviewed experience patch become a persistent service change?

The answer is: only after certification.

The certification forge introduces release candidates, regression packs, golden cases, rollback drills, compatibility deltas, gate decisions, operator review and post-release retention watch.
The forge is intentionally biased against hidden growth. A repeated micro-friction may propose a patch, but the assistant service form changes only when the release ritual completes.
