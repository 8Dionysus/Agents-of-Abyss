# Experience Regression Forge


The regression forge protects the service promise from improvement theater.

Every assistant release candidate must be tested against:
- golden cases: promises that must remain stable,
- boundary cases: scope and escalation discipline,
- degradation cases: graceful failure under missing context/tool loss,
- handoff cases: state packets and receipts,
- negative cases: illegal projection and self-rewrite attempts.

A patch is allowed to improve one behavior only if it does not silently weaken older commitments.
