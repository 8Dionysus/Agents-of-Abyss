# Release Deployment

Release Deployment is an active Experience part. It owns installation, deployment, release train, rollout rings, rollback drills, first governed release, semver, and repository landing order grammar.

## Use when

Use when Experience affects release order, installation posture, deployment readiness, rollout rings, or rollback practice.

## Active outputs

- release gate, installation boundary, rollout ring, rollback trigger, repo landing order, and release ritual route.
- Owner handoff language when the center cannot accept or prove the operational slice.
- Stop-lines that keep Experience center law from becoming live runtime, memory, proof, or owner-local truth.
