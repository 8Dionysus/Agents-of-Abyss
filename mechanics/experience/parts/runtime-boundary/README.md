Runtime Boundary is an active Experience part. It owns living-workspace runtime boundary language, runtime authority resolver constraints, runtime case queues, kind-safe projection, and auto-rollback stop-lines.

## Use when

Use when Experience names runtime-adjacent behavior while keeping activation outside the center repository.

## Active outputs

- runtime stop-line, authority resolver boundary, case queue handoff, projection boundary, and rollback route.
- Owner handoff language when the center cannot accept or prove the operational slice.
- Stop-lines that keep Experience center law from becoming live runtime, memory, proof, or owner-local truth.
