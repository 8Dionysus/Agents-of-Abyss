# Owner Landing

Owner Landing routes reviewed candidates into the repository that owns final
object truth.

## Use When

- A candidate has survived review and needs owner-local acceptance.
- A request packet must be carried into a stronger repository.
- A landed object, reanchor, merge, defer, or drop receipt is needed.

## Do Not Use When

- The center is only drafting doctrine.
- The owner has already landed final truth and only local maintenance remains.
- The route has no reviewable evidence.

## Route Check

Name the candidate, the intended final owner, the requested owner-local surface,
the proof route, and the stop-line that prevents center overclaiming.

## Active Outputs

- Owner request ID.
- Required owner landing.
- Handoff surface hint.
- Owner-local receipt expectation.

## Next Route

Carry the request to the owner repository. Do not mark it accepted or landed in
the center until the owner provides a receipt.
