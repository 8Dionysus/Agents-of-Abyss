# Method Promotion

Method Promotion routes stable repeated scenario work into playbook-owned
method.

## Use When

- A route spans multiple actions, fallback posture, proof hooks, and handoffs.
- The same scenario recurs often enough to need a durable choreography.
- A skill or technique alone cannot carry the full route.

## Do Not Use When

- The object is only a reusable practice.
- The object is only a bounded execution procedure.
- The route is still a one-off donor harvest or experiment.

## Route Check

Ask whether the repeated route is scenario-level, which owners participate, and
what evidence must exist before it becomes a playbook.

## Active Outputs

- Playbook request.
- Scenario boundary.
- Fallback posture hint.
- Required evidence route.

## Next Route

Route recurring scenario method to `aoa-playbooks`, proof to `aoa-evals`, and
component practice or execution pieces to `aoa-techniques` or `aoa-skills`.
