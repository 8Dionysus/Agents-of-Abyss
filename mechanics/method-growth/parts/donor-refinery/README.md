# Donor Refinery

Donor Refinery handles the first conversion from outside pressure or repeated
work into a clean candidate for Method-growth routing.

## Use When

- A donor, repeated workflow, or recurring operator move looks reusable.
- The useful pattern must be separated from foreign doctrine or local noise.
- Provenance matters before technique, skill, or playbook promotion.

## Do Not Use When

- The source owner already has a canonical object.
- The task only needs local cleanup, not growth routing.
- The donor material cannot be summarized without importing its authority.

## Route Check

Ask what reusable pattern survives, which source evidence supports it, which
contamination must be rejected, and which owner should receive the next packet.

## Active Outputs

- Donor summary.
- Reusable-pattern hypothesis.
- Contamination stop-line.
- Next owner route.
