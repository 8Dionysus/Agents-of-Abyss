# Pruning

Pruning keeps failed, merged, deferred, or superseded growth routes explicit
instead of letting them linger as confusing residue.

## Use When

- A candidate should be reanchored, merged, deferred, dropped, or superseded.
- A donor route produced useful lessons but should not become a durable object.
- Retention would create noise unless a clear owner reason exists.

## Do Not Use When

- The owner repository has already retired the object locally.
- The issue is proof failure that belongs first in `aoa-evals`.
- The route should become a playbook, technique, skill, or memory object.

## Route Check

Name the current stage, the reason it should not advance as-is, the owner that
must record the receipt, and the smallest retained lesson.

## Active Outputs

- Prune decision.
- Reanchor or merge target.
- Drop reason.
- Memory or stats routing hint.

## Next Route

Route retention lessons to `aoa-memo`, derived summaries to `aoa-stats`, proof
failures to `aoa-evals`, and local retirement to the object owner.
