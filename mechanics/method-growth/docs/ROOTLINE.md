# AoA x ToS Rootline

This document records the current first-wave rootline for the AoA and ToS ecosystem.

It is a center-layer coordination surface, not the primary home of memory contracts, ToS node schemas, or runtime mechanics.

## Rootline law

The current rootline is:

- AoA acts
- ToS remembers
- alignment and boundaries precede capability growth
- authored/core memory stays distinct from derived retrieval substrate
- ToS grows through node law and lineage before archive expansion
- bridges between AoA and ToS must be explicit, reviewable, and reversible

## First-wave order

The current first planting wave is:

1. ontology spine of action
2. narrative-core memory
3. layered node contract
4. lineage before archive
5. self-agent checkpoint stack
6. interpretation ladder
7. explicit AoA-ToS bridge contracts

Witness traces and context compost are intentionally later.
They become stronger once the structural law above is already explicit.
The rootline establishes checkpoint posture structurally; [RECURRENCE_PRINCIPLE](RECURRENCE_PRINCIPLE.md) names the recovery law for returning from those checkpoints when continuity fails.

## Next wave after the rootline

Once the structural law is explicit, the next wave should move through method rather than jumping straight into heavier protocol pilots.

The second-wave method spine is:

1. method lives in playbooks
2. donor refinement stays source-first
3. practice lineage gains conceptual legitimacy without collapsing into ToS source nodes
4. maturation stays evidence-backed rather than score-driven

See [METHOD_SPINE](METHOD_SPINE.md) for the compact doctrine.

## What this wave should answer

After the first wave, a reader should be able to answer:

- how `technique`, `skill`, `method`, `role`, `memory`, and `eval` differ
- what stays in authored/core memory and what becomes derived substrate
- what a ToS node minimally requires
- why lineage is stronger than flat archival accumulation
- where self-agent surfaces are checkpointed rather than romanticized
- how AoA and ToS exchange bounded experience and retrieval context
