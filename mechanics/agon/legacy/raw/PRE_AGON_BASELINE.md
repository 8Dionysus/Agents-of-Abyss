# Pre-Agon Baseline

This document defines how to name the current system before the Agon imposition begins.

A clean release before Agon is valuable. It is not Agon-readiness.

## Baseline name

Use:

> Pre-Agon Baseline

This phrase means:

- the federation is valid under its current public contracts;
- the center is legible enough to be judged;
- Agon has not yet rechartered actor form, assistant form, protocol law, verdicts, scars, retention, runtime, or ToS promotion;
- all later Agon changes can be compared against this stable pre-turn state.

## What the baseline may claim

The baseline may claim current release cleanliness, current documented ownership boundaries, current generated registry validity, and current public route validity.

It may claim that Agon has a preparation posture if `mechanics/agon/legacy/raw/AGON_PREPARATION_POSTURE.md` exists.

It may not claim that the federation is Agon-ready.

## What to capture before imposition

Before or during Wave 0 landing, capture the current state of:

- release tag or release candidate;
- `README.md`;
- `CHARTER.md`;
- `ECOSYSTEM_MAP.md`;
- `docs/LAYERS.md`;
- `docs/FEDERATION_RULES.md`;
- `mechanics/antifragility/docs/ANTI_AUTHORITY_RULES.md`;
- `mechanics/agon/legacy/raw/AGON_PREPARATION_POSTURE.md`;
- `ROADMAP.md`;
- compact generated center surfaces;
- current validation commands and test state.

## Why the baseline matters

Agon needs a before-image.

Without a named baseline, every future cut looks like drift. With a baseline, every cut can be reviewed as survival, recharter, defer, prune, or quarantine.

## Baseline danger

The baseline must not become a relic shield.

A surface is not protected from Agon because it was present in the baseline. The baseline only proves what existed before the trial.

## One-line rule

Release-clean is not Agon-ready.
