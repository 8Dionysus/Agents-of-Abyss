# Agon Arena Session Model

Wave VIII gives Agon its first session-shaped skeleton.

This is not live protocol. It does not create arena sessions, execute lawful moves, issue verdicts, write scars, schedule retention, mutate rank, or promote anything to `Tree-of-Sophia`.

The purpose of this wave is narrower and sharper: define what a future arena session must look like before any runtime exists.

An arena session model contains a charter, seat requirements, lifecycle slots, pressure profile slots, terminal outcome slots, owner handoffs, and explicit stop-lines.

The model is a vessel, not the blood inside it. Wave IX will later define sealed commit and state packets. Wave XI will later define verdict / delta / scar bridge. Runtime remains later still.
