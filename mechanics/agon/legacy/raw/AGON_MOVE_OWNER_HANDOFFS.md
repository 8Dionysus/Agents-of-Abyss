# Agon Move Owner Handoffs

## Purpose

Wave III keeps the move language center-owned while refusing to absorb neighboring truth.

A future Agon move becomes real only through owner-respecting handoffs.

## Ownership split

`Agents-of-Abyss` owns:

- lawful move vocabulary;
- move class boundaries;
- pre-protocol legality intent;
- owner handoff map;
- stop-lines that prevent drift.

`aoa-techniques` owns:

- reusable practice behind a move;
- portable thinking patterns;
- validation and adaptation methods.

`aoa-skills` owns:

- bounded execution workflows;
- reviewable procedures;
- skill trigger and contract surfaces.

`aoa-routing` owns:

- arena gate hints;
- dispatch toward owner surfaces;
- future `agon_needed` style navigation.

`aoa-playbooks` owns:

- repeated trial choreography;
- mechanical trial scenario packaging;
- campaign-level sequences once the arena exists.

`aoa-evals` owns:

- proof doctrine;
- breach checks;
- lawful outcome assessment;
- verdict bundles.

`aoa-memo` owns:

- durable scars;
- delta history;
- retention checks;
- recall of arena outcomes.

`aoa-stats` owns:

- derived summaries;
- movement signals;
- non-sovereign observability.

`aoa-kag` owns:

- derived survivor structures;
- retrieval-ready lifts;
- no source-authority replacement.

`Tree-of-Sophia` owns:

- source-linked canonization;
- no direct arena write path.

## Handoff pattern

A move record should name possible future owner handoffs, but never implement them.

For example:

```json
{
  "move_id": "agon.move.trace.probe_trace",
  "future_owner_handoffs": {
    "aoa-techniques": ["trace reconstruction practice"],
    "aoa-skills": ["trace probe workflow"],
    "aoa-evals": ["trace integrity proof"]
  }
}
```

This says where later work belongs. It does not create that work.

## Assistant boundary

Assistant service actors may support clarity, receipts, references, escalation, and boundary flags.

They do not gain arena sovereignty through a move handoff.

The legal phrase is:

```text
assistants may help the arena become legible;
assistants may not become hidden contestants.
```

## Stop-line

If a move starts to require execution steps, it is probably becoming a skill.

If it starts to require proof scoring, it is probably becoming an eval.

If it starts to store consequences, it is probably becoming memo.

If it starts to choose scenarios, it is probably becoming playbook.

If it starts to route real actors, it is probably becoming routing.

If it starts to run as durable service, it is probably becoming runtime substrate.

The center keeps the law. It does not devour the city.
