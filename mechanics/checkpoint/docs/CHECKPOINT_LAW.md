# Checkpoint Law

Checkpoint is the lawful intermediate state between active work and stronger
owner truth.

It exists to keep growth from disappearing between work slices, review gates,
returns, closeouts, runtime exports, and owner handoffs.

## Core law

A checkpoint may preserve:

- evidence refs
- owner hints
- review posture
- return anchors
- provisional lineage hints
- local runtime or control-plane receipts
- next-owner pressure

A checkpoint must not mint:

- `candidate_ref`
- `seed_ref`
- memory canon
- proof verdict
- runtime activation
- route authority
- stats truth
- owner acceptance

## Authority ladder

Checkpoint evidence is lower-authority than reviewed closeout.
Reviewed closeout is lower-authority than owner-local landing.
Owner-local landing is lower-authority than proof when proof is required.
Derived summaries remain weaker than all owner-local evidence.

## Honest movement

The honest movement is:

`checkpoint signal -> reviewed checkpoint posture -> closeout bridge or owner handoff -> owner-local landing -> proof when needed`

Shortcuts are allowed only when the route stays smaller than checkpoint
mechanics and does not create a public or owner-local claim.

## Stop-lines

- A raw append log is not doctrine.
- A checkpoint note is not harvest.
- A return anchor is not permission to continue.
- A runtime export is not memory.
- A derived count is not proof.
- A center request is not owner acceptance.
