# Distillation Owner Map

Distillation is center-owned only as law, vocabulary, stop-line, route grammar,
and handoff discipline.

## Center owns

| Surface | AoA center responsibility |
|---|---|
| Distillation law | define what conversion may and may not claim |
| Active extraction discipline | keep functioning parts lighter than source packets |
| Provenance bridge grammar | keep raw routes reachable without making them active law |
| Noise-pruning posture | name what may be removed from active routes and what must stay preserved |
| Candidate handoff | prepare owner-local packets without claiming acceptance |
| Runtime and compost boundaries | prevent runtime packs or ToS compost from being promoted by center wording |

## Stronger owners

| Owner | Owns |
|---|---|
| `aoa-techniques` | reusable distillation techniques and practice canon |
| `aoa-skills` | executable distillation workflows |
| `aoa-playbooks` | recurring checkpoint, session, campaign, or pilot distillation scenarios |
| `aoa-agents` | runtime `distill` phase posture and `distillation_pack` artifact contracts |
| `aoa-memo` | memory writeback candidates and durable memory review |
| `aoa-evals` | provenance-preservation proof and abstraction-quality verdicts |
| `aoa-sdk` | typed helpers, control-panel support, and compatible readers |
| `Dionysus` | seed, donor, and staging preservation |
| `Tree-of-Sophia` | compost, principle, and canon-facing meaning |
| `abyss-stack` | runtime storage, execution, and export plumbing |

## Stop-lines

- Distillation is not summarization.
- Distillation is not proof.
- Distillation is not memory canon.
- Distillation is not runtime activation.
- Distillation is not owner acceptance.
- Distillation is not ToS canon or compost authority.
- Distillation is not raw deletion authority.
- Generated or derived surfaces do not author distillation authority.
