# Provenance Bridge

Use this part to keep the route back to evidence visible without making raw
material the operating surface.

## Use When

- Active docs need to cite older material, sibling evidence, or reviewed source
  packets.
- A future reader must be able to audit what was extracted and what was left
  behind.

## Do Not Use When

- The active surface can stand without source audit.
- The bridge would become a source-file inventory inside active docs.

## Route Check

Point to the controlled bridge, receipt, or index; keep source meaning with the
source owner; avoid direct source catalogs in active parts.

## Active Outputs

- Provenance bridge.
- Receipt or index route.
- Audit-only stop-line.

## Next Route

Use `receipt-index` when repeated source references need compact IDs and
`validation-gate` when provenance preservation must be checked.
