# Receipt Index

Use this part when active surfaces need compact references to older material.

## Use When

- Multiple active artifacts need to cite old packets, staged inputs, or sibling
  owner evidence.
- Direct source paths would pollute active docs.

## Do Not Use When

- One plain provenance link is enough.
- The receipt would pretend to be proof or owner acceptance.

## Route Check

Each receipt must name source route, owner, review state, and allowed use.
Receipts are pointers, not authority.

## Active Outputs

- Receipt ID.
- Source route.
- Allowed-use note.
- Stop-line against promotion.
