# Noise Pruning

Use this part to remove load that makes active routes harder to use.

## Use When

- Active docs contain duplicate history, packet scaffolding, stale route
  wording, raw inventories, or bridge residue.
- A smaller route would preserve meaning better than a larger one.

## Do Not Use When

- The material has not been preserved or routed through provenance.
- The removal would hide unresolved uncertainty.

## Route Check

Name what is pruned, what active function remains, and where evidence survives
if a future audit needs it.

## Active Outputs

- Pruning decision.
- Active-surface cleanup.
- Preservation pointer when needed.

## Next Route

Use `validation-gate` to check that pruning did not erase provenance, owner
boundary, or review state.
