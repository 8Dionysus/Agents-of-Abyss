# Active Extraction

Use this part to move the surviving function into the surface that agents will
actually read and use.

## Use When

- A packet or source contour contains a current rule, route, stop-line, or owner
  handoff that should survive.
- The active route is bloated by historical scaffolding.

## Do Not Use When

- The output would be only a summary with no active function.
- The extracted object belongs directly to a stronger owner and needs handoff
  before center wording.

## Route Check

Name the active surface, the function it owns, what was deliberately left
behind, and which owner keeps final truth.

## Active Outputs

- Active rule or route.
- Functioning part update.
- Source route via provenance bridge.
- Owner stop-line.

## Next Route

Use `noise-pruning` after extraction and `candidate-handoff` when owner-local
truth must land outside the center.
