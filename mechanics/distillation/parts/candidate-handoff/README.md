# Candidate Handoff

Use this part when distilled material should leave the center as an owner-local
request.

## Use When

- Extraction produced a possible technique, skill, playbook, proof, memory,
  runtime, SDK, seed, or ToS object.
- The center can name the candidate but cannot accept it.

## Do Not Use When

- The center still cannot name the owner.
- The candidate lacks source route, review state, or stop-line.

## Route Check

Name the candidate, owner, nearest wrong owner, required landing, proof route,
and stop-line before handoff.

## Active Outputs

- Owner request.
- Handoff packet.
- Rejected nearest-wrong target.
- Next owner route.

## Next Route

Use `OWNER_REQUESTS.md` and carry the request ID into the owner repository.
