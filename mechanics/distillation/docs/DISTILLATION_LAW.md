# Distillation Law

Distillation is the act of converting material into a lighter active form while
preserving the route back to evidence and the boundary to the stronger owner.

## Core law

Distillation must preserve four things:

- source route
- review state
- active function
- owner boundary

If any of those disappear, the result is only a summary, not a reliable
distillation.

## Distillation sequence

1. Identify the source material and its review state.
2. Preserve or route the raw material without making it active law.
3. Extract the surviving function into the active surface that will actually be
   used.
4. Remove packet scaffolding, duplicates, stale claims, and long tails from the
   active route.
5. Keep receipts or provenance bridges when older material must remain cited.
6. Route final object truth to the stronger owner.
7. Validate that no proof, memory, runtime, owner, or canon claim was inflated.

## Distillation is not

- summarization
- proof
- memory canon
- runtime activation
- owner acceptance
- ToS compost or canon authority
- raw deletion authority

## Active-route rule

An active surface should answer the current task directly. It should not make a
future reader traverse raw packets, old waves, transcript residue, or source
inventories unless the task is explicitly provenance audit.

## Provenance rule

Preserved source material remains valuable, but it must not become the default
operating route. Use `PROVENANCE.md`, receipt indexes, or local legacy indexes
as controlled bridges.

## Owner rule

Distillation may prepare a candidate. It does not accept that candidate for the
owner repository.
