# AoA Charter

## Purpose

Agents of Abyss (AoA) is a modular ecosystem for building, evolving, and governing agent-centered knowledge systems over long horizons.

This repository is the canonical high-level statement of that ecosystem.
It exists to keep the overall shape of AoA clear even as implementation spreads across specialized repositories.

## Mission

AoA exists to support sustained intellectual and technical work through:
- reusable techniques
- bounded agent workflows
- portable evaluation surfaces
- routing and recall layers
- modular infrastructure
- human reviewable growth

## What this repository owns

This repository owns ecosystem-level truth about:
- what AoA is
- which public layers belong to AoA
- what role each layer plays
- how the layers relate to one another
- what principles govern growth across the federation

## What this repository does not own

This repository does not own the primary content of specialized layers.

It should not become the main home for:
- technique bundles
- skill bundles
- eval bundles
- memory objects
- agent runtime configurations
- routing datasets
- local infrastructure implementation details

## Core principles

- truth and reproducibility over legend
- modular growth over brittle fusion
- source-of-truth boundaries must stay explicit
- human meaning remains primary
- agents should gain acceleration without inheriting chaos
- new tools should become new layers, not confusion multipliers
- reviewability matters at every layer

## Source-of-truth rule

Specialized repositories own their own meaning.

Examples:
- `aoa-techniques` owns technique truth
- `aoa-skills` owns skill truth
- `aoa-evals` owns proof-surface truth
- `aoa-routing` owns navigation and dispatch surfaces
- `aoa-memo` should own memory and recall truth
- `aoa-agents` should own role and persona truth

This repository owns only the ecosystem-level map and constitution.

## Role in the federation

Treat `Agents-of-Abyss` as:
- the public portal for AoA
- the canonical ecosystem map
- the constitutional center of the AoA federation
- the stable high-level entrypoint for humans and coding agents

## Long-term direction

If AoA matures successfully, this repository should remain compact.
Its job is not to absorb the ecosystem.
Its job is to make the ecosystem intelligible.
