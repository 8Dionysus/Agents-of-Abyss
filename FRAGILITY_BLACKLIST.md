# FRAGILITY_BLACKLIST

This blacklist names patterns that should be removed, merged, narrowed,
suppressed, quarantined, or refused before the ecosystem grows new surface
area.

## 1. Authority drift

A non-owner layer starts sounding canonical.
Default response: shrink it back to a hint, view, recall surface, or link.

## 2. Duplicate meaning

Two or more tracked surfaces say the same thing with slightly different
wording.
Default response: keep one owner surface and merge or delete the rest.

## 3. Hidden mutation

A helper, route, playbook, or runtime surface performs state change without an
explicit reviewable posture.
Default response: stop, expose the mutation boundary, or delete the convenience
wrapper.

## 4. Archived material in active surfaces

Historical, rollback, blocked, or restore-only guidance appears inside active
runbooks or default product docs.
Default response: move it to the archive or delete stale remnants.

## 5. Score sovereignty

A derived or evaluative layer compresses a broad family of signals into one
prestige number.
Default response: split back into vectors, bounded verdicts, or suppressed
summaries.

## 6. Unsourced routing

A route hint or dispatch rule depends on anecdote, memory-only recall, or
vibes.
Default response: require owner receipts or bounded proof, or suppress the
hint.

## 7. Memory as proof

Memo objects speak as if they are the live truth rather than reviewed context.
Default response: downgrade to recall context, add review posture, or remove
the object.

## 8. Derived substrate as source replacement

KAG or other generated projections duplicate source text and then begin to
outrank it.
Default response: quarantine stale packs and delete low-value copies.

## 9. Bundle proliferation

Evals, techniques, skills, playbook lanes, or agent profiles multiply faster
than their distinctions.
Default response: merge near-duplicates and keep only materially different
variants.

## 10. Surface inflation

A repo gains new tracked docs because a thought felt important, not because
ownership demanded a new canonical role.
Default response: merge into an existing canonical surface or keep it local-only.

## 11. Wrapper policy capture

SDK or runtime helpers quietly embed workflow policy that belongs in source
repos, routing registries, or explicit owner docs.
Default response: narrow the helper or remove the wrapper.

## 12. Low-evidence permanence

A weak signal, temporary finding, or thin experimental output becomes a stable
tracked surface.
Default response: suppress, quarantine, or mark expiring.

## 13. Unbounded retry and auto-repair

Runtime or orchestration logic keeps intervening without clear stop conditions
or operator-visible receipts.
Default response: add explicit stop thresholds or remove the self-healing
shortcut.

## 14. Persona inflation

New agent or role surfaces are created for style, flavor, or slightly
different wording rather than real contract change.
Default response: merge profiles and keep one explicit role contract.

## 15. Exception without sunset

A one-off override becomes permanent because nobody remembers why it exists.
Default response: add expiry and review, or delete it.
