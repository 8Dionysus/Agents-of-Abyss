# Agon Move Owner Binding

## Purpose

Wave IV binds the Wave III lawful move language to owner repositories without turning the center into the whole machine.

A lawful move is still a center-owned legal word. Wave IV adds the next layer: each move now declares which neighboring owners must later carry practice, bounded execution, proof, routing, scenario choreography, memory intake, derived observability, or actor-form constraints.

This is not an arena protocol. It is the bridge before the bridge gets weight.

## What changes

Before Wave IV, a move could say:

```text
probe_trace is a lawful pre-protocol move.
```

After Wave IV, the system can also say:

```text
probe_trace keeps legal vocabulary in Agents-of-Abyss,
practice in aoa-techniques,
bounded workflow in aoa-skills,
proof checks in aoa-evals,
memory intake in aoa-memo,
and derived observation in aoa-stats.
```

The move is no longer a floating name. It has owner gravity.

## Ownership law

`Agents-of-Abyss` keeps:

- legal move vocabulary;
- move class boundaries;
- pre-protocol legality;
- owner-binding law;
- stop-lines.

It does not keep:

- reusable practice truth;
- skill workflow truth;
- proof truth;
- durable memory truth;
- route authority;
- scenario choreography;
- stats sovereignty;
- ToS canon truth;
- runtime body.

## Stop-line

Wave IV must not create:

```text
arena sessions
sealed commits
runtime packet envelopes
contradiction ledger storage
verdicts
scar writes
retention scheduling
Tree-of-Sophia promotion
```

Those belong to later waves and owner repositories.

## One-line rule

Wave IV does not make moves executable. It makes their future ownership legible.
