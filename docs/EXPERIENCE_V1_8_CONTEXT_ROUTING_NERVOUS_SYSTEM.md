## Core Law

The v1.8 center contract is a bounded context-routing grammar over the v1.7
candidate consequence layer:

```text
observe task and state
  -> collect context signals
  -> score salience
  -> allocate attention budget
  -> select lawful context layers
  -> build context bundle
  -> guard authority and boundary
  -> dispatch service / arena / memory / proof / dossier candidate
  -> monitor drift
  -> write route receipt
  -> owner landing or recharter request
```

Its laws are:

- No new repository creation.
- No routing without reason, budget, and receipt.
- No routing authorship of owner meaning.
- No salience as authority.
- No Codex context override.
- No assistant context expansion and no personal-context overreach.
- No affect or rank direct rights route.
- No service-to-Agon escalation without lawful trigger.
- No direct `Tree-of-Sophia` write.
- No durable memory write and no retention execution.
- No runtime cache, worker, or hidden scheduler activation.
- No stats, memo, or routing as authority.

## Context Routing Boundary

`context routing` here is a bounded nervous-system contour. It may activate
layers, request review, quarantine stale context, and write receipts. It may
not author the truth of memory, proof, rank, affect, runtime, or canon.

`salience` here is only a prioritization surface. It does not become authority.
`attention budget` is required whenever context expansion would otherwise drift
into hidden pull or unjustified breadth.

Re-entry remains receipt-backed and reviewable. Personal context must stay
materially necessary, minimized, and reversible. Canonical risk may route
toward dossier review, but never directly into `Tree-of-Sophia`.

## Blocking Contracts

The v1.8 center contract treats archive-local packet models, route requests,
route decisions, route receipts, state machines, drift alarms, dashboards,
layer-health rollups, cache records, worker jobs, runbooks, verdict bundles,
memo decisions, KAG nodes, and cross-repo owner proposals as quarantine
evidence unless an owner repository later lands an adapted surface through its
own gate.

The center contract requires:

- landing states remain draft, proposed, reviewed, blocked, quarantined, or
  candidate-only; `active` is not a landing state
- live context routing, live runtime activation, and live owner override are
  forbidden
- route packets require reason, budget, and receipt posture
- routing may not author owner meaning or override center law
- assistant and personal context expansion remain minimized, reviewable, and
  reversible
- service-to-Agon escalation requires lawful trigger review
- canonical risk may route only toward dossier review, not direct canon write
- memo may intake re-entry candidates but may not write durable memory or
  execute retention
- runtime workers, caches, schedulers, jobs, ports, and daemons are forbidden
- dashboards and summaries stay derived and non-authoritative
