# RPG Runtime Projection Wave

## Role

This document defines the first body-facing rollout for the AoA RPG reflection contour.

It is the pass where the contour stops being only a federation of ideas and gains runtime-owned read models, generated transport collections, and a bounded projection seam.

## Goal

Prove that AoA can give the RPG contour a body inside `abyss-stack` without:

- mutating upstream quest meaning
- mutating role, skill, or technique canon
- turning route-api into a write authority
- turning frontend bundles into hidden doctrine
- collapsing AoA and ToS into one runtime mythology

## Included in this wave

### `Agents-of-Abyss`

Owns:
- the common runtime/projection scope note
- the cross-repo wave quest
- federation-level wording about body versus authority

### `abyss-stack`

Owns:
- runtime collection posture
- collection wrappers and generated transport files
- advisory `/rpg/*` seam posture
- frontend projection posture
- builder order and runtime-owned path discipline

### `Dionysus`

Owns:
- prep-pack lineage
- transport note and map
- no doctrinal takeover

## Reused but unchanged in this wave

- `aoa-agents`
- `aoa-evals`
- `aoa-playbooks`
- `aoa-routing`
- `aoa-memo`
- `aoa-sdk`

These layers remain upstream inputs, not new owners of runtime state.

## Important runtime caveat

This wave does **not** require runtime-local mirrors for `aoa-skills` or `aoa-techniques`.

In this pass:
- ability ids may remain pass-through refs
- feat ids may remain pass-through refs
- frontend bundles may cite them without pretending the runtime now owns their canon

That keeps the body honest while the mirror surface is still immature.

## Excluded for now

Deferred, not rejected:

- `8Dionysus`
- standalone frontend code
- a dedicated artifact owner layer
- a reward service
- a reputation daemon
- new service ports
- ToS counterpart overlays in runtime cards

## Success criteria

The runtime/projection wave is successful when:

- `abyss-stack` can publish collection wrappers for builds, ledgers, runs, and bundles
- the collection files can be read by SDK and future frontend code without scraping many repos directly
- route-api can gain a future read-only `/rpg/*` seam without widening authority
- every projection card keeps cited source refs visible
- penalties and reputation remain slice-based and cited rather than mystical

## Failure signs

Pause the contour if:

- runtime collections start claiming source quest state
- bundle cards start awarding progression or changing reputation on their own
- route-api adds write verbs for RPG state
- build snapshots start redefining skill or technique meaning
- ToS language starts replacing canonical runtime keys rather than reflecting them

## Final rule

Let the body carry the contour.

Do not let it rewrite the soul.
