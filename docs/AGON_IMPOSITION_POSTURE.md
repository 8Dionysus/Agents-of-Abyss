# Agon Imposition Posture

This document marks the first Agon turn in `Agents-of-Abyss`.

It does not implement the arena. It does not open a runtime protocol. It does not create a sibling repository. It does something earlier and more severe:

> Agon becomes a criterion by which the current system is judged.

The previous center posture prepared a place for Agon. This posture declares that the existing federation is no longer treated as sacred merely because it is validated, released, documented, or familiar.

From this point, existing surfaces survive by serving lawful conflict, witnessability, actor formation, memory separation, proof separation, runtime restraint, and canon restraint.

## Status

Agon is now an imposed review lens.

Agon is still not a live implementation layer.

Agon is still not a new sibling repository.

Agon remains center-owned as a constitutional protocol contour inside `Agents-of-Abyss`, unless a later reviewed center decision narrows or rehomes a concrete implementation surface.

## The turn

The old question was:

> How can AoA prepare for Agon without breaking the federation?

The new question is:

> Which parts of the federation deserve to survive Agon scrutiny?

This is not license for chaos. It is a stricter law.

Survival is not granted by age. Survival is granted by fitness under Agon.

## The imposition law

A surface may survive when it can show that it strengthens at least one of these conditions without violating the others:

- lawful conflict rather than chat theater;
- explicit actor form rather than vague prompt folklore;
- witnessable claims rather than hidden assertion;
- owner-owned meaning rather than center absorption;
- proof separation rather than evaluator sovereignty;
- memory separation rather than role-profile scar hoarding;
- derived observability rather than score monarchy;
- routing as gate rather than truth narration;
- assistant anti-drift rather than hidden contestants;
- ToS canon restraint rather than direct arena-to-canon write paths;
- runtime restraint rather than premature service mythology.

A surface that cannot show this is not automatically deleted. It is marked for one of five verdicts: survive, recharter, defer, prune, or quarantine.

## Agon survival verdicts

Use these verdicts when auditing current surfaces.

`survive`
: The surface already supports Agon-readiness without changing ownership or smuggling authority.

`recharter`
: The surface has useful material but must be rewritten under Agon law.

`defer`
: The surface is plausible but too early. It must wait for actor formation, protocol law, proof, memory, or runtime owner work.

`prune`
: The surface duplicates authority, blurs ownership, or creates decorative mythology without useful contract.

`quarantine`
: The surface is dangerous if consumed as live truth. Keep it visible only with explicit warning until repaired or removed.
