# QUESTBOOK first wave

## Role

This document defines the first `QUESTBOOK` contour for the AoA foundation pack.
It is a foundation pass, not a new numbered AoA wave.

## Goal

Prove `QUESTBOOK` in the federation without turning every repo into a bureaucracy maze.

## Included in this contour

### `Agents-of-Abyss`

Owns:
- common `QUESTBOOK` model
- first contour scope
- federation-level cross-repo quests only

### `aoa-agents`

Owns:
- quest execution passport
- difficulty, risk, control, and tier mapping
- local-wrapper allowance language

### `aoa-routing`

Owns:
- thin routing over derived quest dispatch hints
- no source quest authorship
- no live parsing of `quests/*.yaml` as authority

### `aoa-playbooks`

Owns:
- harvest thresholds for scenario-shaped repetition
- `reanchor` relation to governed return and valid anchors

### `aoa-memo`

Owns:
- closure evidence writeback boundaries
- quest witness traces that help recall without replacing truth

## Excluded for now

Deferred, not rejected:
- `aoa-sdk`
- source/proof contour repos such as `aoa-techniques`, `aoa-skills`, and `aoa-evals`
- boundary/runtime contour repos such as `aoa-kag`, `abyss-stack`, and `Tree-of-Sophia`
- profile and seed-garden mirrors such as `Dionysus` and `8Dionysus`
- application-level proving grounds until the foundation seams are proven

## Success criteria

The first proof is successful when:
- center quests stay rare and federation-level
- `aoa-agents` defines the execution passport for difficulty, risk, control, and delegation
- `aoa-routing` consumes only derived quest projections and stays thin
- `aoa-playbooks` gets the harvest and `reanchor` seam instead of inheriting backlog drift
- `aoa-memo` gets the evidence writeback boundary without replacing source truth
- expansion stays gated on proof of the foundation seams rather than naming urgency
