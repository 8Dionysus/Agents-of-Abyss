# AbyssOS Organ Contract

This district is the center-owned contract for treating a repository as an
AbyssOS organ.

Use it when a change affects how a repository presents identity, owner
boundary, required route surfaces, surface states, first-cycle work, or
cross-repository handoff posture.

## District law

AoA may define the constitutional shape of an organ. It may name the minimum
surfaces that make an organ reviewable, the surface-state vocabulary that keeps
truth legible, and the first cycle an agent can use to enter without losing
owner boundaries.

Operational helpers, typed control-plane behavior, dispatch, runtime execution,
role state, proof canon, memory objects, and ToS-authored meaning stay with
their stronger owners.

The entry route mode is `organ-alignment` in
[`START_HERE_ROUTE_CONTRACT`](../START_HERE_ROUTE_CONTRACT.md).

Route-mode context: `first-reading`, `root-editing`, `direction-change`,
`ownership-routing`, `mechanic-change`, `organ-alignment`,
`public-claim-validation`, `low-context-agent`, and `district-work` remain
governed by the same route contract.

## Must not claim

- organ alignment is SDK activation
- an organ contract accepts downstream implementation
- a generated capsule owns meaning
- a first cycle replaces local mechanic, release, audit, quest, or checkpoint
  routes
- a public organ claim proves runtime behavior

## Promotion path

Use this district when the center needs a shared organ contract before
descending into sibling repositories.

Promote work from this district only when a stronger owner accepts the next
surface:

| Need | Next owner |
|---|---|
| typed helper, compatibility, activation, or dashboard | `aoa-sdk` |
| dispatch or navigation implementation | `aoa-routing` |
| role posture or agent handoff | `aoa-agents` |
| recurring scenario or campaign composition | `aoa-playbooks` |
| proof bundle or verdict logic | `aoa-evals` |
| memory or recall object | `aoa-memo` |
| graph-ready derived knowledge substrate | `aoa-kag` |
| derived observability | `aoa-stats` |
| runtime service, storage, worker, or deployment | `abyss-stack` |
| ToS-authored meaning | `Tree-of-Sophia` |
