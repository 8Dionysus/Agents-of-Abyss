# Recurrence Manifests

This subdirectory holds recurrence-related manifest surfaces.

Recurrence means bounded return to a valid anchor. It does not mean ambient memory, hidden persistence, autonomous resume loops, or runtime self-healing.

## Use this subdirectory when

- a route needs a reviewable re-entry anchor
- a recurrence adapter or owner handoff needs a manifest
- a future owner-local landing must preserve a bounded return path

## Rules

- Link each manifest to the source doctrine or mechanic branch it supports.
- Do not use recurrence manifests as a memory database.
- Do not grant runtime authority through a manifest.
- Keep owner boundaries explicit.
