# Agents of Abyss (AoA)

Agents of Abyss is an evolving ecosystem of AI agents, reusable techniques, and modular infrastructure for building long-horizon knowledge systems.

## What AoA is

AoA is not a single bot or a narrow automation project.

It is a multi-agent architecture designed to help grow, maintain, and operationalize complex knowledge systems over time. Its agents handle routine and technical work such as collection, cleaning, transformation, validation, linking, documentation, and infrastructure support.

AoA is designed to remain extensible: new techniques, new tools, and new agent roles should become new layers of the system rather than one-off accidents.

## Why it exists

Most AI projects solve isolated tasks.

AoA exists to support continuous intellectual and technical work:
- growing large knowledge systems
- turning reusable practices into explicit techniques
- connecting humans and agents through reviewable workflows
- building infrastructure that can evolve without collapsing into chaos

## Relationship to Tree of Sophia

AoA is the operational and agentic side of the ecosystem.

Tree of Sophia (ToS) is the living knowledge architecture that AoA helps build, maintain, and expand.

In short:
- **AoA** = agents, techniques, workflows, infrastructure
- **ToS** = knowledge graph, texts, concepts, lineages, interpretation layers

## Current status

This repository is currently a public landing and direction marker.

The core implementation is being staged through related repositories and will be expanded after the modular refactor of `abyss-stack`.

## Near-term direction

Planned public layers include:
- reusable agent techniques
- modular local and hybrid AI infrastructure
- workflow patterns for agent-assisted engineering
- knowledge operations for long-horizon projects
- bridges between operational agents and knowledge architectures

## Principles

- truth and reproducibility over legend
- human meaning, agent acceleration
- modular growth over brittle monoliths
- explicit source of truth
- reviewable and safe workflows
- new tools become new layers, not chaos multipliers

## For contributors and coding agents

If you are a coding agent or contributor, treat this repository as the canonical high-level statement of AoA.

Implementation details may currently live in adjacent repositories until the modular public structure is finalized.

When in doubt:
- preserve the distinction between agent operations and knowledge architecture
- prefer reusable techniques over project-specific hacks
- prefer modularity over premature fusion
- preserve clear links between AoA, ToS, and `abyss-stack`
